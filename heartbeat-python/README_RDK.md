# rPPG心率检测系统 - RDK摄像头版本

## 项目简介

这是一个基于rPPG(远程光电容积脉搏波)技术的实时心率检测系统,支持RDK板子摄像头和普通USB摄像头。系统通过Flask提供Web服务,将视频流和心率数据实时推送到网页端展示。

## 主要特性

- ✅ **双摄像头支持**: 自动检测并使用RDK硬件加速摄像头,如不可用则降级到普通OpenCV摄像头
- ✅ **实时心率检测**: 使用POS算法从面部视频中提取心率信号
- ✅ **Web可视化**: 通过浏览器实时查看视频流和心率数据
- ✅ **自适应FPS**: 动态计算实际帧率并调整算法参数
- ✅ **CORS支持**: 允许跨域访问API接口

## 配置说明

### config.py 参数

```python
VIDEO_SOURCE = 0          # 摄像头ID (仅对普通摄像头有效)
SAMPLING_FREQUENCY = 30   # 采样频率(Hz)
RESCAN_FREQUENCY = 10     # 人脸重新检测频率(每N帧)
MIN_SIGNAL_SIZE = 100     # 信号处理最小窗口大小
MAX_SIGNAL_SIZE = 300     # 信号处理最大窗口大小
```

### RDK摄像头参数

在 `main.py` 的 `HeartbeatMonitor.__init__()` 中可以修改:

```python
self.cap = RDKCamera(
    camera_id=0,   # 摄像头ID
    width=640,     # 分辨率宽度
    height=480,    # 分辨率高度
    fps=30         # 目标帧率
)
```

## API接口

### 1. 视频流

```
GET /video_feed
```

返回MJPEG格式的视频流,可直接在`<img>`标签中使用。

### 2. 心率状态

```
GET /api/status
```

返回JSON格式的心率数据:

```json
{
  "bpm": 75.5,
  "count": 45,
  "status": "online"
}
```

- `bpm`: 最近3秒的平均心率
- `count`: 参与计算的有效数据点数量
- `status`: 设备状态 ("online" 或 "searching")

### 3. 药品识别(预留)

```
GET /api/drug
```

当前返回空数据,预留用于未来扩展。

## 技术细节

### RDK摄像头工作原理

1. **YUV NV21格式**: RDK摄像头输出YUV NV21格式数据
2. **自动分辨率检测**: 根据数据大小自动推断实际分辨率(支持1920x1080、1280x720、640x480等)
3. **格式转换**: 将YUV转换为BGR供OpenCV处理
4. **硬件加速**: 利用RDK芯片的ISP处理能力

### rPPG算法流程

1. **人脸检测**: 使用Haar级联分类器检测人脸
2. **ROI提取**: 提取额头区域作为感兴趣区域
3. **信号提取**: 计算ROI的平均RGB值
4. **去趋势处理**: 使用移动平均消除光照变化
5. **POS算法**: 投影到颜色空间分离血流信号
6. **带通滤波**: 滤除42-210 BPM范围外的噪声
7. **频域分析**: FFT计算心率主频
8. **时域验证**: 峰值检测交叉验证

## 常见问题

### Q1: 提示"hobot_vio库未找到"

**A**: 这是正常现象。如果你在普通PC上运行,系统会自动切换到普通摄像头模式。如果要在RDK板上使用,请确保已正确安装`hobot-vio`库。

### Q2: 网页无法显示视频流

**A**: 检查以下几点:
1. 确认IP地址是否正确(将HTML中的`192.168.43.216`改为实际IP)
2. 确认防火墙是否允许5000端口
3. 确认程序是否正常启动(查看控制台输出)

### Q3: 心率检测不准确

**A**: 参考以下优化建议:
1. **光照条件**: 确保充足均匀的光照,避免频闪
2. **距离适中**: 面部距离摄像头30-50cm最佳
3. **保持静止**: 测量时头部尽量保持不动
4. **额头暴露**: 确保额头区域无遮挡(刘海、帽子等)
5. **等待稳定**: 程序启动后需等待10-15秒积累足够数据

### Q4: FPS过低导致心率不准

**A**: 
- RDK摄像头固定为30fps,无需担心
- 普通摄像头可通过降低分辨率提升帧率
- 系统会自动计算真实FPS并调整算法

## 性能优化建议

1. **使用RDK摄像头**: 硬件加速可提供稳定的30fps
2. **降低分辨率**: 640x480已足够,更高分辨率会增加处理负担
3. **关闭自动曝光**: 固定曝光参数可减少光照噪声
4. **自然光照明**: 避免50/60Hz频闪的人造光源

## 调试技巧

### 查看实时日志

程序会在控制台输出关键信息:
- FPS更新通知
- 信号处理错误
- 摄像头初始化状态

### 验证算法正确性

使用**手指测试**:用手指直接接触摄像头,此时应能检测到较高的心率值(因为信号很强)。如果此时准确,说明算法逻辑正确,问题可能出在面部ROI质量或环境因素上。
